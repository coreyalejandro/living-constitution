What "shipped" means

A roadmap is only useful if "shipped" has a strict definition. For The Living Constitution:

  • An artifact is shipped when it is published at its stated path, content-hashed and (where applicable) signed, with at least one external reader able to access it without credentials.
  • An artifact is reproducible when an external developer can verify its claims using the publicly stated procedure (e.g., make verify) within 90 seconds of clone.
  • An artifact is operational when a non-author user has used it for the intended purpose at least once.

The Published table contains shipped artifacts. The Reproducible / Operational gates are tracked in /inventory.md.

Quarterly amendment

This roadmap is revised at every quarterly amendment cycle. Changes are append-only at the row level; corrections are added as new rows with notes. Version history is at the bottom of this file (currently v1.0).

The honesty discipline of this roadmap is the most important property it has. If a row moves from In-Progress to Published, that is celebrated. If a row slips its target by more than one quarter, that is published with the same prominence as a success.
